# Recommended way to create a new API in this structure

Follow this flow:

1. **Define the endpoint**
2. **Add or update migration** in `migrations/` if schema changes
3. **Add SQL query in `internal/db/query/`**
4. **Generate DB code with `sqlc`**
5. **Add business logic in `internal/service/`**
6. **Add handler in `internal/http/`**
7. **Register route in router**
8. **Test with curl/Postman**

Note:
- For docker container, to apply new API, re-build without cache (`make apply-new-api`)

---

# Example: create `GET /v1/rooms/{id}`

---

## 1) Decide responsibility of each layer

Using this structure:

- `**cmd/api/`**
  - app startup
  - dependency wiring
  - router creation
- `**internal/http/**`
  - parse request
  - validate request input
  - call service
  - return JSON response
- `**internal/service/**`
  - business logic
  - orchestration
  - validation and rules
  - transaction handling if needed
- `**internal/db/**`
  - DB connection
  - `sqlc` query files
  - `sqlc` generated code
- `**migrations/**`
  - schema changes

---

## 2) Suggested internal structure

```text
internal/
  http/
    handlers/
      room_handler.go
    routes/
      routes.go
    middleware/
  service/
    room_service.go
  db/
    postgres.go
    query/
      room.sql
    dbgen/            # generated by sqlc
```

> **Note**: For this project, prefer `**sqlc` + service layer** first.  
> Add a repository layer only if the codebase grows enough to justify the extra abstraction.

# Step-by-step example

## 3) Add DB query with `sqlc`

Create `internal/db/query/room.sql`:

```sql title="internal/db/query/room.sql"
-- name: GetRoomByID :one
SELECT id, status, created_at, updated_at
FROM rooms
WHERE id = $1
LIMIT 1;
```

---

## 4) Generate code with `sqlc`

Run:

```bash
make sqlc
```

or:

```bash
sqlc generate
```

This will generate strongly typed Go code in:

```text
internal/db/dbgen/
```

---

## 5) Create service layer

### `internal/service/room_service.go`

```go title="internal/service/room_service.go"
package service

import (
	"context"

	"backend-service/internal/db/dbgen"
)

type RoomService struct {
	queries *dbgen.Queries
}

func NewRoomService(queries *dbgen.Queries) *RoomService {
	return &RoomService{
		queries: queries,
	}
}

func (s *RoomService) GetRoomByID(ctx context.Context, id int64) (*dbgen.Room, error) {
	room, err := s.queries.GetRoomByID(ctx, id)
	if err != nil {
		return nil, err
	}

	return &room, nil
}
```

---

## 6) Create HTTP handler

### `internal/http/handlers/room_handler.go`

```go title="internal/http/handlers/room_handler.go"
package handlers

import (
	"encoding/json"
	"net/http"
	"strconv"

	"backend-service/internal/service"

	"github.com/go-chi/chi/v5"
)

type RoomHandler struct {
	roomService *service.RoomService
}

func NewRoomHandler(roomService *service.RoomService) *RoomHandler {
	return &RoomHandler{
		roomService: roomService,
	}
}

func (h *RoomHandler) GetRoomByID(w http.ResponseWriter, r *http.Request) {
	idParam := chi.URLParam(r, "id")
	id, err := strconv.ParseInt(idParam, 10, 64)
	if err != nil {
		http.Error(w, "invalid room id", http.StatusBadRequest)
		return
	}

	room, err := h.roomService.GetRoomByID(r.Context(), id)
	if err != nil {
		http.Error(w, "room not found", http.StatusNotFound)
		return
	}

	w.Header().Set("Content-Type", "application/json")
	w.WriteHeader(http.StatusOK)
	_ = json.NewEncoder(w).Encode(room)
}
```

---

## 7) Register routes

### `internal/http/routes/routes.go`

```go title="internal/http/routes/routes.go"
package routes

import (
	"net/http"

	"backend-service/internal/http/handlers"

	"github.com/go-chi/chi/v5"
)

func RegisterRoutes(r chi.Router, roomHandler *handlers.RoomHandler) {
	r.Get("/healthz", func(w http.ResponseWriter, r *http.Request) {
		w.WriteHeader(http.StatusOK)
		w.Write([]byte("ok"))
	})

	r.Route("/v1", func(r chi.Router) {
		r.Get("/rooms/{id}", roomHandler.GetRoomByID)
	})
}
```

---

## 8) Wire everything in `cmd/api/main.go`

### `cmd/api/main.go`

```go title="cmd/api/main.go"
package main

import (
	"context"
	"log"
	"net/http"
	"os"
	"time"

	"backend-service/internal/db"
	"backend-service/internal/db/dbgen"
	"backend-service/internal/http/handlers"
	"backend-service/internal/http/routes"
	"backend-service/internal/service"

	"github.com/go-chi/chi/v5"
	"github.com/go-chi/chi/v5/middleware"
)

func main() {
	port := os.Getenv("PORT")
	if port == "" {
		port = "8080"
	}

	databaseURL := os.Getenv("DATABASE_URL")
	if databaseURL == "" {
		log.Fatal("DATABASE_URL is required")
	}

	ctx := context.Background()

	pool, err := db.NewPool(ctx, db.Config{DatabaseURL: databaseURL})
	if err != nil {
		log.Fatalf("db connect failed: %v", err)
	}
	defer pool.Close()

	queries := dbgen.New(pool)

	roomService := service.NewRoomService(queries)
	roomHandler := handlers.NewRoomHandler(roomService)

	r := chi.NewRouter()
	r.Use(middleware.RequestID)
	r.Use(middleware.RealIP)
	r.Use(middleware.Recoverer)
	r.Use(middleware.Timeout(5 * time.Second))

	routes.RegisterRoutes(r, roomHandler)

	log.Printf("listening on :%s", port)
	log.Fatal(http.ListenAndServe(":"+port, r))
}
```

# Example for `POST /v1/rooms`

If you want a **create API**, the flow is similar:

- handler parses JSON body
- service validates business rules
- `sqlc` query inserts into DB

---

## 9) Add SQL query

### `internal/db/query/room.sql`

```sql title="internal/db/query/room.sql"
-- name: CreateRoom :one
INSERT INTO rooms (status)
VALUES ($1)
RETURNING id, status, created_at, updated_at;
```

Then regenerate:

```bash
make sqlc
```

---

## 10) Add request model in handler

### `internal/http/handlers/room_handler.go`

```go
type CreateRoomRequest struct {
	Status string `json:"status"`
}
```

Add handler method:

```go title="internal/http/handlers/room_handler.go"
func (h *RoomHandler) CreateRoom(w http.ResponseWriter, r *http.Request) {
	var req CreateRoomRequest
	if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
		http.Error(w, "invalid request body", http.StatusBadRequest)
		return
	}

	room, err := h.roomService.CreateRoom(r.Context(), req.Status)
	if err != nil {
		http.Error(w, err.Error(), http.StatusBadRequest)
		return
	}

	w.Header().Set("Content-Type", "application/json")
	w.WriteHeader(http.StatusCreated)
	_ = json.NewEncoder(w).Encode(room)
}
```

---

## 11) Add service logic

### `internal/service/room_service.go`

```go title="internal/service/room_service.go"
func (s *RoomService) CreateRoom(ctx context.Context, status string) (*dbgen.Room, error) {
	if status == "" {
		status = "inactive"
	}

	room, err := s.queries.CreateRoom(ctx, status)
	if err != nil {
		return nil, err
	}

	return &room, nil
}
```

---

## 12) Register POST route

### `internal/http/routes/routes.go`

```go title="internal/http/routes/routes.go"
r.Route("/v1", func(r chi.Router) {
	r.Get("/rooms/{id}", roomHandler.GetRoomByID)
	r.Post("/rooms", roomHandler.CreateRoom)
})
```

# Recommended convention to follow

## Handler

Handler should contain only HTTP concerns:

- path params
- query params
- request body parsing
- response formatting
- HTTP status codes

---

## Service

Service should contain business logic:

- input validation
- business rules
- orchestration across queries
- transaction handling
- permission checks

---

## DB / `sqlc`

DB layer should contain SQL only:

- `SELECT`
- `INSERT`
- `UPDATE`
- `DELETE`

Do not put business logic into SQL files.

---

# Naming convention

## Files

Prefer **domain/resource-based names**:

- `room_handler.go`
- `room_service.go`
- `room.sql`

Avoid naming things directly after DB tables unless the domain name is the same.

---

## Methods

Prefer **use-case/action names**:

- `GetRoomByID`
- `CreateRoom`
- `UpdateRoom`
- `DeleteRoom`

For more complex business flows, use names like:

- `CreatePlayerAndAssignRoom`
- `RegisterPlayer`
- `AssignPlayerToRoom`

---

## SQL query names

Use precise DB action names:

- `GetRoomByID`
- `CreateRoom`
- `UpdateRoomStatus`
- `DeleteRoom`

These names become generated Go methods in `sqlc`.

---

## Routes

Use REST-style resource names:

- `GET /v1/rooms/{id}`
- `POST /v1/rooms`
- `PATCH /v1/rooms/{id}`
- `DELETE /v1/rooms/{id}`

---

# Recommended workflow for every new API

## For read API

1. **Add SQL query** in `internal/db/query/`
2. **Run `sqlc generate`**
3. **Add service method**
4. **Add handler**
5. **Register route**
6. **Test with curl**

---

## For write API

1. **Add migration** if schema changes
2. **Apply migration**
3. **Add SQL query** in `internal/db/query/`
4. **Run `sqlc generate`**
5. **Add service validation/business rules**
6. **Add handler**
7. **Register route**
8. **Test with curl/Postman**

---

# When to add a repository layer

By default, you can let the service use `dbgen.Queries` directly.

Add a repository layer only if:

- services become too coupled to generated code
- multiple services share the same persistence logic
- mocking/testing becomes awkward
- you want a stricter domain boundary

Until then, `**sqlc + service` is enough**.

# My recommendation for this project

For now, use this convention:

```text
internal/http/handlers/
internal/http/routes/
internal/service/
internal/db/query/
internal/db/dbgen/
```

This is simple, scalable, and works very well for a Go service using **PostgreSQL + `sqlc`**.

---

## Optional note for more complex APIs

If an API involves multiple steps, such as:

- create player
- randomly assign room
- update room capacity

then:

- keep **HTTP** in handler
- keep **workflow/business logic** in service
- keep **individual SQL operations** in `sqlc`
- use a **transaction** in service if multiple writes must succeed together
